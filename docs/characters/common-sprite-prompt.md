# 캐릭터 공통 이미지 에셋 생성 프롬프트 — 런타임 규격 (v2)

> 인게임 스프라이트를 **런타임 규격 그대로** 생성하기 위한 공통 프롬프트.
> 기존 등급 문서의 ①/② 블록(대형 시트 → 절단 스크립트 정규화)과 달리,
> 이 규격은 최종 파일 치수로 바로 뽑는다. 유닛별 외형 묘사는 각 등급 문서(`1-common.md` ~ `6-transcendent.md`)에서 가져와 뒤에 이어붙인다.

## 공통 규격 (절대 규칙)

### 크기

| 항목 | 값 |
| --- | --- |
| 시트 전체 | **576×144px** — 가로 1행 4프레임 |
| 프레임 | **144×144px** 균등 간격 |
| 캐릭터 높이 | **132px 고정** (프레임의 92%) — 모든 프레임·모든 유닛 동일 |
| 캐릭터 폭 | 동작에 따라 가변 (기준: 75 → 97 → 97 → 94px) |
| 정렬 | 발끝이 프레임 바닥에서 **6px 위 고정** (바닥 기준선 통일), 좌우 중앙 정렬 |

### 비율

- 두신비 **약 1:3.5 세미치비** — 머리가 몸 대비 크고 팔다리는 짧게 데포르메 (전 유닛 공통)
- 프레임 내 가로세로비 **0.57(팔 모은 포즈) ~ 0.73(팔 뻗은 포즈)** — 동작이 커져도 폭만 늘고 키는 132px 유지
- 4프레임 균등 간격, **프레임 0(첫 프레임) = 대기 포즈 겸용** (`art.ts` 규칙)
- 공중 유닛: 발 위치 대신 **부유 높이 고정 + 발밑 분리 타원 그림자**

### 디자인

- **32비트풍 픽셀아트** — 32px급 캐릭터 도트 밀도로 디자인하고 132px 높이로 출력 (도트가 굵게 살아 있어야 함)
- 어두운 외곽선, 채도 낮은 차분한 색감, 디더링 최소
- 4프레임 공격 사이클: 준비 → 장전/조준 → 발동(임팩트) → 마무리
- **공격 방향별로 좌향/우향 시트를 각각 생성** — 단순 미러 금지 (무기 파지·소품 방향·비대칭 디테일이 뒤집히면 안 됨)

---

## [공통 스타일 블록] — 모든 프롬프트의 맨 앞

```
32비트 시대의 정교한 픽셀아트, 32px급 캐릭터 도트 밀도(굵은 도트),
채도 낮은 차분한 색감, 어두운 외곽선, 디더링 최소.
캔버스 576×144px — 가로 1행 4프레임, 각 프레임 144×144px 균등 간격.
배경은 완전 투명(알파), 글자·로고·팔레트 칩 없음.

【크기 통일 — 시리즈 공통 규격, 반드시 지킬 것】
- 캐릭터 높이(발끝~정수리)는 모든 프레임에서 132px 고정 (프레임의 92%).
- 발끝은 프레임 바닥에서 6px 위의 기준선에 고정, 좌우는 프레임 중앙 정렬.
- 두신비 약 1:3.5 세미치비 — 어떤 유닛과 나란히 놓아도 키·머리 크기가 같아 보여야 한다.
- 동작이 커져도 키는 그대로, 폭만 변한다 (가로세로비 0.57~0.73 범위).
- 동물·기계·부유형도 같은 높이 규격을 채운다 — 크기감은 부피·실루엣으로 표현.
- 공중 유닛은 발 위치 대신 부유 높이를 고정하고 발밑에 분리된 타원 그림자를 넣는다.
```

## [공격 모션 블록 — 우향 ➡] (스타일 블록 뒤에 이어붙임)

```
오른쪽을 향한 옆모습으로 [해당 유닛의 공격]을 하는 4프레임 공격 모션 시트.
프레임 구성: ①준비(대기 겸용) → ②장전/조준 → ③발동(임팩트) → ④마무리.
무기·소품·가방 등 비대칭 디테일의 위치가 4프레임 내내 일관되어야 한다.
투사체가 있으면 ④에서 캐릭터와 분리된 별도 픽셀로 그린다.
```

## [공격 모션 블록 — 좌향 ⬅] (스타일 블록 뒤에 이어붙임)

```
왼쪽을 향한 옆모습으로 [해당 유닛의 공격]을 하는 4프레임 공격 모션 시트.
우향 시트의 단순 좌우 반전이 아니라, 무기 파지와 소품(가방·완장·번호 등)의
방향이 실제로 왼쪽을 향하도록 다시 그린다.
프레임 구성·높이·기준선 규칙은 우향 시트와 완전히 동일.
```

---

## 사용법

1. [공통 스타일 블록] + [공격 모션 블록(우향 또는 좌향)] + **유닛별 캐릭터 묘사·공격 컨셉**(등급 문서의 ①② 참조) 순으로 이어붙여 생성한다
2. 한 유닛당 공격 시트 2장(우향/좌향)을 뽑는다 — 파일명 규칙(안): `{id}_attack_right_sheet.png` / `{id}_attack_left_sheet.png`
3. 걷기(앞/뒤/옆) 시트도 같은 스타일·크기 블록을 쓰고, 모션 블록의 방향·동작 문구만 교체한다

> ⚠ 코드 현황: `src/data/art.ts`는 현재 공격 시트 1장(우향) + 좌측 미러 방식이다.
> 좌향 전용 시트를 도입하려면 `UnitArt.attack`에 좌/우 분리 필드를 추가하고
> `Unit.ts`의 미러 로직을 방향별 시트 선택으로 바꿔야 한다 (📝 미구현).

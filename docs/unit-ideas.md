# 유닛 확장 아이디어 — 100종 로스터 & 전설 페어 초월

> 아이디어 문서. 확정 기획이 아니며, 채택分은 GDD 파트 1과 `src/data/units.ts`로 옮긴 뒤
> `node --experimental-strip-types scripts/balance-sim.ts`로 검증한다.

## 1. 목표와 구조

- **안흔함~전설 100종**: 기존 6계열(24종) + 신규 19계열(76종) = **25계열 × 4등급 = 100종**
- 흔함 등급은 기존 6계열 6종을 유지 (뽑기 입문용 — 계열 수를 늘리지 않음)
- 초월: 기존 레시피 5종 유지 + **전설 페어 초월 13종** 신규 (아래 4장)
- 설계 원칙은 GDD와 동일 — 계열은 테마·역할의 일관성, 랜덤성은 뽑기·합성이 담당

## 4. 전설 페어 초월 — "연관성 있는 조합"

기존 "전설 2기 → 랜덤 초월" 천장을 확장: **지정 전설 페어는 두 전설의 테마를 계승한 지정 초월**이 된다.
페어가 아닌 전설 2기는 기존대로 랜덤 초월 (천장 유지). 기존 재료 3기 레시피 초월 5종도 유지.

| 초월 | 페어 (전설+전설) | 연관성 | 역할 |
| --- | --- | --- | --- |
| 시티 로드 | 시장 + 경찰청장 | 행정과 치안, 도시 권력의 정점 | 물리 광역 + 전 유닛 공격력 오라 |

## 5. 밸런스 경고 & 도입 전략

**GDD 1.6의 교훈**: v2에서 전설 풀을 3→6으로 늘리자 레시피 완성률이 급락해 봇 승률 57%→11%로 붕괴했다.
전설 풀을 6→25로 늘리면 특정 전설 2기를 모으는 난이도가 그대로 폭증한다. 그래서:

1. **한 판에 전체 풀을 쓰지 않는다** — 게임 시작 시 25계열 중 10~12계열만 활성화 (고정 프리셋 or 판마다 랜덤). 원랜디의 "시즌 풀"에 해당
2. 페어 초월은 **활성 계열 안에서만** 등장하도록 페어를 역할군별로 분산 배치
3. 랜덤 천장(비페어 전설 2기 → 랜덤 초월)은 반드시 유지 — v2.1을 살린 안전장치
4. 도입은 계열 단위로 단계적: 서포트/오라군(방송·요리·금융) → 딜러군 → 유틸군 순으로 4~5계열씩 추가하며 매 단계 `balance-sim.ts` 재검증 (봇이 오라·경제를 모르면 시뮬레이터 봇도 함께 보강)
5. 뽑기 확률은 계열 수가 아니라 **활성 풀 크기** 기준으로 재조정

## 6. 조합 중심 확장안 — 흔함 10종 × 3기 조합 = 220 콘텐츠

> 3~5장의 계열/등급안과 별개 축의 아이디어. 계열표(3장)는 조합 결과 유닛의
> 컨셉·아키타입 풀로 재활용한다. 핵심 철학: **콘텐츠 물량과 성장 도파민.**

### 6.1 구조

- 흔함 유닛 풀 **10종** = 일반 8종 + **히든 2종** (뽑기에서만 낮은 확률로 등장, 시작 선택 불가)
- 게임 시작 시 일반 8종 중 **3종 선택** → 히든 2종과 합쳐 판당 활성 흔함 **5종**
- 합성: 흔함 **3기 조합 (중복 허용)** → 결과 유닛. 전체 조합 C(12,3) = **220가지 전부에 결과 배정**
- 시작 덱 선택지: C(8,3) = **56가지** — 어떤 선택이든 모든 220 조합은 언젠가 도달 가능 (죽은 콘텐츠 없음)

### 6.2 판당 콘텐츠 수 (정확한 수치)

활성 흔함 5종으로 만들 수 있는 3기 중복 조합 = C(7,3) = **35가지**:

| 형태 | 계산 | 수 |
| --- | --- | --- |
| AAA (3기 동일) | 5 | 5 |
| AAB (2+1) | 5×4 | 20 |
| ABC (모두 다름) | C(5,3) | 10 |

- 판당 등장 가능 유닛: 흔함 5 + 조합 결과 35 = **40종** (전체 220의 ~16%) → 판마다 다른 조합 트리 = 리플레이 동력
- 35조합 중 히든 재료가 필요 없는 것은 C(5,3) = 10개뿐. **나머지 25개는 히든 유닛이 재료** — 히든이 뽑기 잭팟이자 레어 레시피의 열쇠

### 6.3 밸런스 — 수치 중복 허용 (확정)

- 220종 전부 고유 수치일 필요 없음. **아키타입 템플릿 ~10종**(연타/한방/광역/마법/감속/방깎/오라/DoT/크리/유틸) × 등급 배율로 배정, 같은 수치의 스킨 차이 유닛 허용
- 고유 메커니즘은 엄선 20~30종만
- 결과 등급은 **히든 재료 수(0/1/2)에 비례해 상향** — 히든 2기 조합 = 최고 등급 잭팟. "업그레이드된 캐릭터가 강해질수록"의 도파민 커브를 여기서 만든다

### 6.4 작명 — 코믹 도심 컨셉

직업에 한정하지 않고 도심에서 볼 수 있는 평범한 것들을 코믹하게:
비둘기 떼, 길고양이, 포장마차, 마을버스, 전단지 아저씨, 헬스 관장, 취객, 폐지 어르신, 눈사람, 캡스 출동차 …

- AAA(10) → 시그니처 진화: 붕어빵 사장 ×3 = "붕어빵 푸드트럭 제국"
- AAB(90) → 지배 재료가 정체성, 부재료가 수식어: 신입 경관 2 + 취객 = "주취자 전담반"
- ABC(120) → 크로스오버: 비둘기 조련사 + 공원 관리인 + 신입 경관 = "공원 순찰연합"

| 전투 스타일 | 떼(스웜) — 약한 광역 쪼기 | 암습 — 단일 크리티컬 할퀴기 |
| 흔함 대비 | 수치는 흔함급, 메커니즘 티저만 부여 | 〃 |
| 진짜 가치 | 전투력이 아니라 **레시피 열쇠** (판당 35조합 중 25개의 재료) | 〃 |

선정 이유:
1. **작명 무한 확장** — 어떤 직업과 섞어도 코믹 크로스오버가 저절로 나온다
2. 낮/밤·하늘/골목·떼/단독의 완전 대비 → 히든끼리도 정체성 충돌 없음
3. 한국 도시 밈(닭둘기, 길냥이 집사) 정합성 — 설명 없이 웃긴다
4. 히든 2기 조합(판당 13개, 최고 등급 구간)이 "앙숙 콤비물"이 된다

레시피 예시:
- 닭둘기 ×3 = **비둘기 대군단** (하늘을 덮는 광역)
- 냥이 ×3 = **골목 냥아치 파벌** (연속 암습)
- 닭둘기 ×2 + 배달 라이더 = **전서구 특급배송** (초고속 연타)
- 냥이 ×2 + 신입 경관 = **K-냥 수사대** (크리 표식)
- 닭둘기 + 냥이 + 취객 = **새벽 광장의 삼파전** (광역 CC)
- 닭둘기 + 냥이 + 임의 재료 = "톰과 제리" 시리즈 — 도감 하이라이트 구간

검토했다 접은 대안 (기록용):
- 로또 1등 당첨자 + 은퇴한 전설의 고수 — "숨은 강자" 트로프는 좋으나 여전히 사람이라 일반 8종과 시각 구분이 약함
- 눈사람 + 소나기 구름 — 계절물 한정 이벤트 히든으로 재활용 여지

### 6.7 낮/밤 시스템

필드가 **낮 ↔ 밤 3분 간격**으로 전환되고, 낮/밤 태그를 가진 유닛이 해당 페이즈에 버프를 받는다.

**사이클**
- 전환 주기 3분. 단, 실제 시계가 아니라 **가상 게임 시계(gameTime) 기준** — 배속(x2/x3)·일시정지와 자동 연동 (이미 구현된 시계 재사용, 추가 비용 없음)
- 40라운드 ≈ 20분이므로 판당 6~7회 전환. 전환 30초 전 HUD에 예고 아이콘 (☀→🌙)

**버프 규칙 — 버프만 있고 페널티는 없음**
- ☀ 낮 태그: 낮 동안 공격력 +25%
- 🌙 밤 태그: 밤 동안 공격력 +25%
- 무태그: 영향 없음. 반대 페이즈 디버프는 두지 않는다 — "내 유닛이 약해졌다"는 체감은 스트레스만 남기고, "지금 얘가 강해졌다"는 같은 밸런스로 도파민만 남긴다

**태그 부여**
- 히든: 닭둘기 = ☀, 골목대장 냥이 = 🌙 (설정 그대로)
- 조합 결과는 히든 재료의 태그를 상속: 닭둘기 포함 조합 = ☀, 냥이 포함 = 🌙, 둘 다 포함 = ☀🌙 (양쪽 페이즈 모두 버프 — 히든 2기 조합의 프리미엄 강화)
- 일반 직업 유닛에도 소수 부여 가능: 야간 배달, 새벽 인력, 유흥가 순찰 등 밤이 어울리는 컨셉 위주

**전략 레이어**
- 덱 3종 선택 + 히든 편중에 따라 "낮 덱/밤 덱"이 생기고, 강한 페이즈에 보스가 걸리도록 라운드 페이스를 조절(배속 활용)하는 플레이가 나온다
- 옵션(후순위): 밤에 몹 이동속도 +10% 같은 몹 측 변화 — 우선 유닛 버프만으로 시작

**구현 비용 (참고)**
- 페이즈 판정: `gameTime % 360000 < 180000` 한 줄
- 버프: 데미지 계산에 태그 배율 한 줄
- 연출: 카메라 배경색 틴트 전환 + HUD 아이콘 — 신규 에셋 불필요
- balance-sim 봇도 페이즈 배율만 반영하면 검증 가능
